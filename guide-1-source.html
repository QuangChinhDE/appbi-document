Chương 1 · Tạo Data Source

Điểm khởi đầu của mọi báo cáo. Bạn sẽ kết nối AppBI tới nơi dữ liệu đang nằm — cơ sở dữ liệu, kho phân tích đám mây, Google Sheets hoặc file Excel/CSV — để hệ thống truy vấn trực tiếp (live), không phải chép tay.

1.1Màn hình Data Sources — nơi quản lý mọi kết nối

Là gì: trang Data Sources liệt kê toàn bộ nguồn đã kết nối. Đây là nơi bạn tạo mới, tìm kiếm, lọc theo loại, kiểm tra kết nối, chia sẻ và xoá nguồn.

⚙ Cách mở & đọc màn hình

  1. Ở thanh điều hướng trái, bấm biểu tượng ổ cắm 🔌 (Data Sources).
  2. Trên cùng là các thẻ thống kê: số kết nối, số loại nguồn, số Sheets / Manual.
  3. Dùng ô Search để tìm theo tên, hoặc bộ lọc All source types để lọc theo loại (PostgreSQL, BigQuery…).
  4. Mỗi dòng hiển thị: tên nguồn, nhãn loại (PostgreSQL/BigQuery/Google Sheets…), chủ sở hữu, ngày cập nhật/tạo, và cột Actions.
  5. Ở cột Actions có 3 nút: Test connection (biểu tượng tia — kiểm tra kết nối), Share (chia sẻ), Delete (xoá).
  6. Góc trên phải có nút chuyển xem dạng lưới / dạng bảng.
Mẹo: đặt tên nguồn theo nghiệp vụ (ví dụ “Olist Postgres”, “DW Buoi 8”) để về sau khi tạo Dataset bạn chọn đúng nguồn ngay, không phải nhớ host/IP.

1.2Tạo nguồn mới & 5 loại nguồn được hỗ trợ

Là gì: AppBI hỗ trợ 5 loại nguồn: PostgreSQL, MySQL, BigQuery, Google Sheets và Manual Table (tải file Excel/CSV). Sau khi bấm tạo, hệ thống tự kiểm tra kết nối ngay.

⚙ Các bước chung

  1. Bấm + New Data Source (Tạo nguồn dữ liệu) ở góc trên phải.
  2. Nhập Name (tên nguồn, bắt buộc *).
  3. Chọn Type (loại nguồn) — phần cấu hình bên dưới sẽ đổi theo loại bạn chọn.
  4. (Tuỳ chọn) nhập Description để đồng đội hiểu nguồn dùng cho việc gì.
  5. Điền phần Connection Configuration (xem chi tiết từng loại bên dưới).
  6. Bấm Create. Dòng chữ “Connection will be checked automatically…” nghĩa là kết nối sẽ được test tự động — nếu sai, hệ thống báo lỗi ngay tại đây.

a) PostgreSQL / MySQL — cơ sở dữ liệu quan hệ

Điền các trường kết nối tiêu chuẩn:

  • Host (ví dụ localhost hoặc IP/tên miền máy chủ) và Port (PostgreSQL mặc định 5432, MySQL 3306).
  • Database (bắt buộc) — tên database cần đọc.
  • Schema (tuỳ chọn, mặc định public với PostgreSQL).
  • Username và Password — bấm Show để xem lại mật khẩu vừa nhập.

b) BigQuery — kho phân tích của Google

BigQuery có 2 cách xác thực, chọn 1 trong 2 ở phần Authentication:

  • Use my Google account — dùng tài khoản Google đã kết nối sẵn trong AppBI (nhanh, hợp cho cá nhân).
  • Use service account — dán Service Account JSON (khoá dịch vụ). Hợp cho môi trường vận hành/chia sẻ đội nhóm vì không phụ thuộc tài khoản cá nhân.

Sau đó nhập Project ID (ví dụ my-gcp-project); nếu chọn service account thì dán nội dung JSON vào ô Service Account JSON.

Lưu ý bảo mật: Service Account JSON được mã hoá khi lưu. Khi sửa nguồn, để trống ô này nghĩa là giữ nguyên khoá cũ (không cần dán lại).

c) Google Sheets — bảng tính

⚙ Cách làm

  1. Chọn Type = Google Sheets, kết nối tài khoản Google.
  2. Chọn spreadsheet và sheet cần dùng.
  3. Bấm Create — dữ liệu được đọc trực tiếp từ Google Sheets mỗi lần xem.

d) Manual Table — tải file Excel / CSV

Khi dữ liệu chỉ nằm trong file, chọn Manual Table rồi kéo-thả file vào vùng tải lên (hoặc bấm để chọn file).

  • Hỗ trợ định dạng .csv, .xlsx, .xls.
  • File Excel nhiều sheet sẽ được nạp tất cả các sheet thành nhiều bảng.
Mẹo cho demo Olist: bộ dữ liệu mẫu Olist đã được nạp sẵn qua nguồn PostgreSQL “Olist Postgres” — các chương sau sẽ dùng chính nguồn này để dựng Dataset & biểu đồ.

1.3Kiểm tra kết nối & sửa cấu hình

Là gì: sau khi tạo, bạn có thể kiểm tra lại kết nối bất cứ lúc nào và cập nhật thông tin (đổi mật khẩu, đổi host…) một cách an toàn.

⚙ Cách làm

  1. Ở danh sách, bấm nút Test connection (biểu tượng tia) trên một nguồn để kiểm tra nhanh mà không cần mở ra.
  2. Muốn sửa: bấm vào tên nguồn để mở chi tiết, rồi bấm Edit.
  3. Sửa các trường cần thiết. Mật khẩu/khoá để trống = giữ nguyên giá trị cũ.
  4. Bấm Update/Save — kết nối được kiểm tra lại tự động; nếu lỗi sẽ báo ngay.
Lưu ý: nếu Test connection báo lỗi, kiểm tra lần lượt: host/port có đúng & máy chủ có mở cho AppBI truy cập không, tên database/schema, và tài khoản có quyền đọc không.

1.4Chia sẻ & phân quyền nguồn

Là gì: cấp quyền cho người/nhóm khác dùng nguồn với 3 mức Viewer (Xem) / Editor (Sửa) / Full (Toàn quyền). Cơ chế chia sẻ này dùng nhất quán cho cả Dataset, Chart, Dashboard và Workboard — nắm ở đây là hiểu toàn hệ thống.

⚙ Cách làm

  1. Bấm biểu tượng Share trên nguồn cần chia sẻ.
  2. Nhập email người dùng (hoặc chọn nhóm/team).
  3. Chọn vai trò Viewer / Editor / Full.
  4. Bấm Share. Có thể thu hồi quyền bất cứ lúc nào trong cùng hộp thoại.

1.5Xong Chương 1 — Checklist & lỗi thường gặp

Trước khi sang Chương 2 (Dataset), hãy chắc chắn:

  • ✅ Đã tạo được ít nhất 1 nguồn và Test connection báo thành công.
  • ✅ Nguồn có tên dễ hiểu theo nghiệp vụ.
  • ✅ Bạn biết nguồn này chứa những bảng nào mình sẽ cần (không cần lấy hết cả schema).
Nhắc lại nguyên tắc: AppBI không sao chép dữ liệu — mỗi lần xem báo cáo, hệ thống truy vấn trực tiếp lên nguồn. Vì vậy chỉ cần giữ kết nối & quyền đọc là dữ liệu luôn mới.
Lỗi hay gặp: ① sai port/host → không kết nối được; ② tài khoản thiếu quyền đọc schema; ③ với BigQuery, thiếu Project ID hoặc dán nhầm Service Account JSON; ④ file Excel sai định dạng ngày/tháng khiến cột bị nhận nhầm kiểu — sẽ xử lý ở Chương 2 (Dataset · Quality).

➡️ Tiếp theo: Chương 2 · Tạo Dataset — gom các bảng từ nguồn này thành một Dataset sẵn sàng để phân tích.
